# 노드 실측 사양표

이 문서는 실제 장비 기준으로 서버/Jetson/Raspberry Pi의 CPU, RAM, GPU/NPU 모델명을 정리한 문서다.
저장소 안의 논리적 역할 정보와, 장비에서 직접 수집한 실측 정보를 분리해서 기록하도록 구성했다.

사용 목적:
- 논문 본문과 실험 환경 표 정리
- 발표 자료용 하드웨어 요약
- placement profile과 실제 장비 스펙의 정합성 확인

작성 원칙:
- 저장소에서 이미 확인된 항목과 실측이 필요한 항목을 구분한다.
- 모르는 값은 추정하지 말고 `미확인`으로 둔다.
- 실측값은 수집 일시와 수집 명령을 같이 남긴다.

실측 기준:
- 수집 일시: `2026-04-10`
- 수집 위치: `etri-ser0001-CG0MSB`에서 직접 확인 + `kubectl debug node/...` 기반 원격 확인
- 수집 방식: `lscpu`, `free -h`, `nvidia-smi`, `lspci`, `hostnamectl`, `kubectl get node -o json/jsonpath`, `/proc/device-tree/model`

## 3. 저장소 기준으로 이미 확인된 항목

아래 정보는 저장소 문서/코드에서 이미 확인되는 항목이다.

| 노드 | hostname | 역할 | 아키텍처 | 가속기 타입 | 메모리 등급 | 계산 등급 | 출처 |
|---|---|---|---|---|---|---|---|
| 서버 1 | `etri-ser0001-CG0MSB` | `cloud_server` | `amd64` / `x86_64` | `gpu_server` 또는 `gpu_discrete` | `high` | `high` | `docs/architecture.md`, `workflow_reporter/workflow_reporter/demo_workflow.py`, `placement_engine/tests/test_engine.py` |
| 서버 2 | `etri-ser0002-CGNMSB` | `cloud_worker` | `amd64` | 미확인 | 미확인 | 미확인 | `docs/architecture.md` |
| Jetson | `etri-dev0001-jetorn` | `edge_ai_device` | `arm64` / `aarch64` | `gpu_embedded` | 문서상 `low`, 테스트상 `medium` | `medium` | `docs/architecture.md`, `workflow_reporter/workflow_reporter/demo_workflow.py`, `placement_engine/tests/test_engine.py` |
| Raspberry Pi 5 | `etri-dev0002-raspi5` | `edge_light_device` | `arm64` / `aarch64` | `none` | `low` | `low` | `docs/architecture.md`, `workflow_reporter/workflow_reporter/demo_workflow.py` |

주의:
- Jetson의 `memory_class`는 문서와 테스트 값이 다르므로, 실측 후 하나로 통일하는 것이 좋다.
- 서버 2는 GPU Operator 잔재를 정리한 뒤 `nvidia-container-toolkit + k8s-device-plugin` 최소 경로로 GPU 노출을 복구했다.
- 서버 1은 containerd v2 계열 설정에서 `enable_cdi = true`를 반영한 뒤 `nvidia.com/gpu=1`로 정상 노출됐다.

## 4. 이번에 사용한 확인 명령 예시

### 서버 1에서 직접 실행

```bash
lscpu
free -h
hostnamectl
lspci | grep -Ei 'vga|3d|display|nvidia'
nvidia-smi --query-gpu=name,memory.total,driver_version --format=csv,noheader
```

### 클러스터에서 각 노드 기본 정보 확인

```bash
kubectl get nodes -o wide
kubectl get node <node-name> -o json
kubectl get node <node-name> -o jsonpath='{...}'
```

### 원격 노드 내부를 직접 확인할 때 사용

```bash
kubectl debug node/<node-name> --image=busybox -- chroot /host sh -lc '<command>'
```

예시:

```bash
kubectl debug node/etri-dev0002-raspi5 --image=busybox -- chroot /host sh -lc 'cat /proc/device-tree/model; lscpu; free -h'
kubectl debug node/etri-dev0001-jetorn --image=busybox -- chroot /host sh -lc 'cat /proc/device-tree/model; lscpu; free -h; dpkg -l | grep -E "nvidia-l4t|jetpack"'
kubectl debug node/etri-ser0002-cgnmsb --image=busybox -- chroot /host sh -lc 'lscpu; free -h; lspci | grep -Ei "vga|3d|display|nvidia"; nvidia-smi || true'
```

## 5. 장비에서 추가 수집할 명령 예시

아래 명령은 각 장비에서 직접 실행해서 표를 채우기 위한 기본 수집 세트다.

### 공통 Linux 기본 수집

```bash
hostnamectl
uname -a
uname -m
cat /etc/os-release
lscpu
free -h
lsblk -o NAME,MODEL,SIZE,TYPE,MOUNTPOINT
ip -br addr
```

### x86 서버에서 GPU 확인

```bash
lspci | grep -Ei 'vga|3d|display|nvidia'
nvidia-smi
```

만약 `nvidia-smi`가 없으면:

```bash
lspci -nn | grep -Ei 'vga|3d|display'
```

### Jetson에서 GPU/플랫폼 확인

```bash
cat /proc/device-tree/model
uname -m
free -h
dpkg -l | grep -E 'nvidia-l4t|jetpack'
tegrastats --help
```

Jetson에서 GPU 상태를 조금 더 보려면:

```bash
sudo /usr/bin/jetson_clocks --show
```

### Raspberry Pi 5에서 플랫폼 확인

```bash
cat /proc/device-tree/model
uname -m
free -h
vcgencmd version
vcgencmd get_config arm_boost
```
